BEFORE (Original PathSafetyAssessmentTool):
--------------------------------------------------------------------------------
Location: backend/app/services/gis_mapping.py (lines 1275-1410)
Size: ~140 lines of inline code

Implementation:
  ├─ Convert point to metric CRS
  ├─ Load 3 path shapefiles via LayerStore
  ├─ For each layer:
  │   ├─ Check CRS and convert if needed
  │   ├─ Remove Z-coordinates (inline method)
  │   ├─ Filter valid geometries
  │   ├─ Standardize WIDTH column (inline method)
  │   └─ Build spatial index
  ├─ Expanding ring search (manual loop):
  │   ├─ Create buffer at each radius
  │   ├─ Query spatial index
  │   ├─ Find candidates with valid WIDTH
  │   ├─ Calculate distances
  │   ├─ Lock first width found
  │   └─ Continue to max_radius
  └─ Categorize width (>4: Wide, >2: Narrow, <=2: Very Narrow)

Helper Methods (inline):
  - _remove_z_coordinate() (35 lines)
  - _standardize_width_column() (40 lines)

Caching:
  - LayerStore caching only

Features:
  - Basic expanding ring search
  - Simple geometry validation
  - Width extraction only
  - No geometry merging

AFTER (Migrated from PathAssignmentTool):
--------------------------------------------------------------------------------
Location: backend/app/services/gis_mapping.py (lines 1275-1333)
Size: ~60 lines (calls utility module)

Implementation:
  ├─ Convert point to metric CRS
  ├─ Extract base_dir from LayerStore paths
  ├─ Call get_radius_and_width_at_point() utility function
  └─ Categorize width (>4: Wide, >2: Narrow, <=2: Very Narrow)

Utility Module: backend/app/utils/path_width_curvature.py (388 lines)
  ├─ load_layer() with file mtime caching
  ├─ remove_z() - comprehensive 3D to 2D conversion
  ├─ standardize_width_column() - 12+ column variants
  ├─ _densify_linestring() - for smoother curvature
  ├─ _min_triplet_radius_from_linestring() - Heron's formula
  ├─ _merge_connectable_lines() - unary_union + linemerge
  ├─ _extract_lines() - handle all geometry types
  ├─ _min_radius_within_window_for_layer() - local curvature
  ├─ _nearest_radius_and_width_with_priority() - core search
  ├─ get_radius_and_width_at_point() - public API (returns R, W)
  └─ get_width_at_point() - backward-compatible API

Caching:
  - File modification time-based cache
  - Layer-level caching
  - Shared cache across multiple calls

Features:
  - Advanced expanding ring search with priority layers
  - Sophisticated geometry processing (merge, densify, repair)
  - Returns BOTH radius and width
  - Curvature calculation (triplet-based with Heron's formula)
  - Comprehensive Z-coordinate removal
  - 12+ WIDTH column variants recognized
