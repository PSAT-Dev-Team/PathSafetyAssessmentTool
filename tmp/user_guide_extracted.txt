PSAT User Guide
Path Safety Assessment Tool (PSAT) User Guide
Welcome to PSAT. The tool helps you create projects from geotagged survey images, review coded path segments, calculate safety risk scores, analyse safety hazards and test treatments.
1. Getting Started
Open the project list
Use the Home page to browse all projects. You can:
search by project name or road name
filter by tag
sort by verification, autocode progress, and last modified time
select one or more projects for coding, path analysis, or treatment work
Create a project
Use Create Project when you want to build a new project from the source image folders in `in/`.
You can create a project in threeways:
select one source folder directly
draw a polygon or click a planning area on the map and create a project from multiple selected roads
upload area GIS layers
When creating a project:
enter a project name without underscores
add tags if needed (for easy grouping e.g. NSC, AMK)
upload images into a source folder if the folder is missing
check that selected roads are marked as available before you create the project
Navigate between workflows
From the project list, you can send selected projects to:
Coding for detailed review and saving
Analyse Projects for multi-project filtering, charts, and exports
Treatment Application for before/after scenario testing
Use Help and GIS Layers
use the Help button for the in-app user, administration and developer guides
use View GIS Layers from the sidebar when you need to inspect the GIS data behind the coding rules

2. Coding Page
The Coding page is the main review workspace. It can open one or more selected projects in a combined session.
Main layout
The page keeps three views in sync:
the current segment image
the attributes table
the segment map
Selecting a segment in one area updates the others.
User can use the following ways to navigate through the segments:
indicate the segment number and jump straight to the segment
click next or back to choose the segment
click the point from the segment map
Default values of the attributes
Auto-code options
PSAT supports several auto-code paths:
CV auto-code from the image
GIS auto-code from the segment geometry
Logic rules
bulk auto-code across selected rows or the full project
per-attribute auto-code in workflows that target only certain attributes
Autocode updates are tracked and displayed in the project listing as Percentage Segments Autocoded, and the Segments Autocoded counter is updated in the project metadata.
Manual review
You can override any coded value directly in the table. The page also shows:
risk score updates (on-the-fly) for the selected segment
boxed-up attributes being overwritten manually
a validation summary table comparing percentage of attributes being over written against the stored baseline (autocoded)
field-source provenance for auto-coded changes
Details and GIS context
For supported attributes, the page can show extra spatial detail such as:
nearby GIS layers (e.g. cycling path, footpath, MRT stations, Bus stops etc) 5m radius around the current segment
curvature visualization
width visualization
grade or gradient details when profile data is available
Save and progress tracking
After review:
save your attribute edits to persist them and recalculate scores
update the Segments Verified Percentage counter as you complete manual checks

3. Map View & Analysis
Map views appear in several PSAT workflows, especially Coding, Path Analysis, Treatment, and GIS Layers.
Segment map in Coding
On the Coding page, the map helps you:
click a segment to select it
see risk-band coloring for saved scoring results
view nearby GIS context for the current point
inspect the spatial position of the current segment
Risk colors
Segments are color-coded by their highest relevant risk band so you can quickly spot high-risk areas.
GIS context
Depending on the page and workflow, PSAT can show nearby (within 5m radius of segment) GIS information such as:
paths and crossings
bus stops and MRT-related features
road-name and planning-area context
width, curvature and gradient analysis overlays
GIS Layers page
Use the dedicated GIS Layers page when you want to inspect the raw shapefile layers themselves rather than just the map context around a segment.
That page lets you preview available layers, review their metadata, and update GIS files when needed.

4. Path Analysis
The Path Analysis page is the multi-project analysis workspace.
Select projects
You can select one or more projects to analyze together. The search box also matches source-road names, so you can find projects even when the project title and road name differ.
Filter before loading
Before loading projects, you can filter by:
project or road name
tag
date created range
last updated range
Analyze loaded data
After loading projects, you can:
choose up to five attributes to focus on
apply segment-level filters
view attribute distributions and aggregated score-band summaries
inspect filtered segments on the map and in the table
upon selecting, the page will navigate to coding page to allow user to make edits (if necessary)
once done, you can click “Back to Analysis” to continue other path analysis
Export
The page supports two main exports:
Download Table for the current filtered table as CSV
Download Images for the current filtered images as a ZIP file
Session continuity
Your selections and filters are kept for the browser session, so navigating away and returning does not immediately clear the analysis setup.

5. Treatment Application
The Treatment workflow lets you test how standard interventions affect risk before you commit changes.
Start from the project picker
Use the Treatment page to search for projects by project name or road name, then load one or more projects into the treatment workspace.
Work by segment or by treatment
Inside the detailed treatment view, you can switch between:
By Segment to review the current segment and its applicable treatments
The treatments are sorted in descending order of how much risk score changes
By Treatment to stack treatments 
The treatments are sorted in descending order by how manyrisk bands are improvedacross the loaded data
Preview and apply
You can:
preview the effect of treatments (i.e. change in risk score/band) before saving
apply selected treatment(s) to one segment
apply one treatment across all applicable segments
apply all applicable treatments in bulk
Save or reset
Treatment changes can be reviewed before they are finalized. Use the page controls to:
save the treatment state
reset pending treatment changes
compare before and after scores while deciding what to keep

GIS Layer Management Guide
PSAT uses external GIS shapefiles and GeoJSON layers to provide context for path coding and risk analysis. Use the GIS Layer Management tool to keep these datasets up to date.
1. Viewing GIS Layers
Click the View GIS Layers button in the sidebar to open the GIS Layers dashboard. Here you can:
Browse all current layers grouped by category (e.g., `area_type`, `bus_lane`).
View metadata for each layer, including the geometry type and feature count.
**Required Columns**: Inspect the mandatory column names needed for PSAT to process the layer. The number in parentheses, e.g., `LU_DESC (1)`, indicates the expected column index in the source data.
2. Adding a New GIS Layer
Use the Add GIS Layer workflow to upload entirely new datasets.
**New Category**: You must provide a name for the category folder (e.g., `school_zones`). Every upload is assigned to a specific category.
**File Upload**: Drag and drop your GIS files. For shapefiles, ensure you upload all companion files together (`.shp`, `.shx`, `.dbf`, `.prj`).
**Preview**: Once uploaded, you can preview the geometry on the map before finalizing.
3. Replacing an Existing GIS Layer
Use the Replace GIS Layer workflow when you have updated data for an existing layer.
**Filterable Search**: Use the searchable inputs to quickly find the folder and specific layer you wish to replace. Simply type a few letters (e.g., "bus") to filter the list.
**Safety Checks**: PSAT performs compatibility checks to ensure the new file has the same required columns as the original.
**Warnings**: If differences are found in the column structure, the system will warn you before overwriting the old data.
